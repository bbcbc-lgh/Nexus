## 1. 项目概述

News APP 前端是一个移动端优先的新闻阅读 SPA（单页应用），最大宽度限制为 480px，适配手机屏幕。主要功能包括：

- 用户注册 / 登录 / 登出
- 按分类浏览新闻，支持无限滚动加载
- 新闻详情阅读，含相关新闻推荐
- 收藏 / 历史记录管理
- 用户资料编辑与密码修改

所有数据通过 REST API 与后端（FastAPI）交互，认证方式为 Token（存储于 localStorage）。

## 4. 路由设计

**配置文件：** `src/router/index.ts`  
**路由模式：** HTML5 History（`createWebHistory`）

### 路由表

| 路径 | 名称 | 组件 | 是否公开 | 说明 |
|------|------|------|----------|------|
| `/login` | login | LoginView | 是 | 登录页 |
| `/register` | register | RegisterView | 是 | 注册页 |
| `/` | - | - | - | 重定向至 `/news` |
| `/news` | news | NewsView | 否 | 新闻列表主页 |
| `/news/detail/:id` | newsDetail | NewsDetailView | 否 | 新闻详情页 |
| `/profile` | profile | ProfileView | 否 | 个人中心 |

### 导航守卫

```ts
router.beforeEach((to) => {
  const token = localStorage.getItem('token')
  if (!to.meta.public && !token) return '/login'
})
```

未登录用户访问受保护路由时，自动跳转至 `/login`。所有路由组件均使用**懒加载**（`() => import(...)`）。

## 5. 状态管理

使用 **Pinia** 管理全局状态，共两个 Store。

### 5.1 authStore（认证状态）

**文件：** `src/stores/authStore.ts`

**State：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `token` | `string \| null` | 认证 Token，初始从 localStorage 读取 |
| `userInfo` | `UserInfo \| null` | 当前登录用户信息 |

**Getters：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `isLoggedIn` | `boolean` | 是否已登录（computed） |

**Actions：**

| 方法 | 说明 |
|------|------|
| `login(username, password)` | 调用登录接口，成功后保存 token 和用户信息 |
| `register(username, password)` | 调用注册接口，成功后自动登录 |
| `fetchInfo()` | 刷新用户信息 |
| `updateInfo(data)` | 更新用户资料 |
| `logout()` | 清除 token 和用户信息，移除 localStorage |

Token 通过 `localStorage.setItem('token', t)` 持久化，刷新页面后自动恢复登录状态。

---

### 5.2 newsStore（新闻状态）

**文件：** `src/stores/newsStore.ts`

**State：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `categories` | `Category[]` | 新闻分类列表 |
| `activeCategoryId` | `number` | 当前选中分类 ID |
| `newsList` | `NewsItem[]` | 当前分类的新闻列表 |
| `page` | `number` | 当前分页页码 |
| `hasMore` | `boolean` | 是否还有更多数据 |
| `loading` | `boolean` | 加载状态标志 |

**Actions：**

| 方法 | 说明 |
|------|------|
| `loadCategories()` | 加载分类列表（有缓存则跳过） |
| `loadNews(categoryId, reset?)` | 加载新闻，`reset=true` 时重置列表 |
| `setCategory(id)` | 切换分类并重新加载新闻 |

## 6. API 接口对接

### 6.1 HTTP 客户端

**文件：** `src/api/client.ts`

基于原生 `fetch` 封装，统一处理请求头、认证 Token 和响应格式。

**响应格式约定：**
```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

当 `code !== 200` 时抛出错误，错误信息取自 `message` 字段。

**认证方式：** 请求头携带 `Authorization: <token>`。

---

### 6.2 接口列表

#### 用户接口（`src/api/user.ts`）

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/user/login` | 否 | 登录，返回 token + userInfo |
| POST | `/api/user/register` | 否 | 注册，返回 token + userInfo |
| GET | `/api/user/info` | 是 | 获取当前用户信息 |
| PUT | `/api/user/update` | 是 | 更新用户资料（nickname/bio/gender/phone/avatar） |
| PUT | `/api/user/password` | 是 | 修改密码 |

#### 新闻接口（`src/api/news.ts`）

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/news/categories` | 否 | 获取分类列表 |
| GET | `/api/news/list?categoryId=&page=&pageSize=` | 否 | 分页获取新闻列表 |
| GET | `/api/news/detail?id=` | 否 | 获取新闻详情（含相关推荐） |

#### 收藏接口（`src/api/favorite.ts`）

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/favorite/check?newsId=` | 是 | 检查是否已收藏 |
| POST | `/api/favorite/add` | 是 | 添加收藏 |
| DELETE | `/api/favorite/remove?newsId=` | 是 | 取消收藏 |
| GET | `/api/favorite/list?page=&pageSize=` | 是 | 获取收藏列表 |
| DELETE | `/api/favorite/clear` | 是 | 清空收藏 |

#### 历史记录接口（`src/api/history.ts`）

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/history/add` | 是 | 添加浏览记录 |
| GET | `/api/history/list?page=&pageSize=` | 是 | 获取历史列表 |
| DELETE | `/api/history/delete/:id` | 是 | 删除单条历史 |
| DELETE | `/api/history/clear` | 是 | 清空历史记录 |

## 7. 页面设计

### 7.1 LoginView — 登录页

- 表单字段：用户名、密码
- 提交后调用 `authStore.login()`，成功跳转 `/news`
- 含加载中 spinner 防重复提交
- 底部提供注册页链接

### 7.2 RegisterView — 注册页

- 表单字段：用户名、密码、确认密码
- 客户端校验：密码最少 6 位，两次输入一致
- 提交后调用 `authStore.register()`，成功直接进入首页
- 底部提供登录页链接

### 7.3 NewsView — 新闻列表页

- 顶部：吸顶分类 Tab 栏，支持横向滚动，选中高亮
- 列表第一条以**大卡片（Hero Card）**展示（带封面图 + 文字覆盖层）
- 后续条目为**标准卡片**（缩略图右置，标题 + 作者 + 时间 + 阅读量）
- 滚动至底部自动触发加载更多（无限滚动）
- 加载时显示**骨架屏动画**（shimmer 效果）
- 全部加载完显示"已无更多"提示
- 时间显示为相对时间（`timeAgo()` 工具函数）

### 7.4 NewsDetailView — 新闻详情页

- 顶部固定栏：返回按钮 + 收藏切换按钮（心形图标）
- 封面大图
- 文章标题、作者、发布时间、阅读量
- 正文内容（HTML 渲染）
- 相关新闻推荐列表（可点击跳转）
- 进入页面自动调用 `/api/history/add` 记录浏览历史
- 含加载中和加载失败状态处理

### 7.5 ProfileView — 个人中心页

- 用户头像（显示用户名首字母）、昵称、用户名
- 编辑资料按钮（弹出底部 Sheet 模态框）
- 修改密码按钮（弹出底部 Sheet 模态框）
- 登出按钮
- Tab 切换：**收藏** / **历史**
  - 两个 Tab 均支持分页加载更多
  - 每条记录支持单独删除
  - 右上角支持"清空全部"
  - 空状态展示提示图标

### 8.2 字体规范

| 字体 | 用途 | 权重 |
|------|------|------|
| Noto Serif SC | 标题、强调文字 | 400 / 600 / 700 / 900 |
| Noto Sans SC | 正文、界面文字 | 400 / 500 / 700 |

回退顺序：PingFang SC → Hiragino Sans GB → Microsoft YaHei → sans-serif

### 8.3 布局规范

- 最大宽度：`480px`，居中显示
- 底部导航栏：固定定位，`backdrop-filter: blur` 毛玻璃效果
- 内容区底部留 `padding-bottom: calc(var(--nav-h) + 16px)` 避免被导航遮挡

### 8.4 动画规范

| 动画名 | 参数 | 用途 |
|--------|------|------|
| `spin` | 0.6s linear infinite | 加载 spinner 旋转 |
| `shimmer` | 1.5s infinite | 骨架屏扫光效果 |
| `bounce` | 1s infinite | 加载更多点动画 |
| Modal 过渡 | fade + slide-up | 底部弹出模态框 |

## 9. 构建配置

### 9.1 Vite 配置（`vite.config.ts`）

```ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8100',
      changeOrigin: true
    }
  }
}
```

- 路径别名：`@` → `./src`
- 开发代理：`/api/*` 转发至后端 `http://localhost:8100`，解决跨域

### 9.2 TypeScript 配置

- 编译目标：ES2020
- 模块解析：bundler 模式
- 严格模式：全量开启（`strict: true`，`noUncheckedIndexedAccess: true`）
- Vue JSX 支持：`jsxImportSource: "vue"`
